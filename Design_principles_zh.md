# 个人网站首页设计原则

基于 Anthropic / Claude 的设计语言研究，但不直接复制。

英文 AI 实现参考见 `Design_principles.md`。

## 1. 总体方向

这个个人网站首页不应该做成“AI 科技感作品集”，而应该更像一个温暖、克制、有思想密度的个人编辑空间。

Anthropic 和 Claude 最值得借鉴的不是米色背景、橙色按钮或极简导航这些表层元素，而是背后的判断：

- 用温暖和留白建立信任。
- 用清晰结构降低认知负担。
- 用克制界面为思考、写作和协作让路。
- 用编辑感、纸质感和手作痕迹，把技术表达变得更有人味。

首页核心方向：

> 一个像“纸质工作台”的个人首页：温暖、安静、清晰，有杂志感，也有 Claude 式的思考空间感。

## 2. 背景：人本安全的 AI 品牌

Anthropic 的品牌根基是人本安全。它强调可靠、可解释、可控制的 AI 系统，并把安全与伦理放在核心位置。

这会直接影响视觉语言：

- 品牌不能显得冰冷、炫技或压迫。
- 页面要表达“人类价值、温暖、可信度”。
- 设计要帮助用户放松下来，相信这里是一个可以安全思考的地方。
- 技术感应该被“阅读、研究、判断、协作”的语境包裹起来。

对个人网站的启发：

- 不要急着证明自己很厉害，先让人觉得可信。
- 不要用复杂视觉制造权威，应该用清晰结构和诚实表达建立信任。
- 技术能力可以展示，但气质应该更像“可靠的合作者”，而不是“闪亮的展示橱窗”。

## 3. Anthropic 品牌身份：Geist 的视觉体系

Anthropic 从隐匿状态进入公众视野时，与波特兰设计工作室 Geist 合作构建了完整视觉形象。这个系统的重点是同时表达“人本使命”和“技术工艺”。

### 3.1 简洁字标与编码符号

Geist 选择了非常纯粹的文字标志，并保留一个斜杠细节，用来暗示代码、计算和未来方向。

对个人网站的启发：

- Logo 或个人标识不需要复杂图形。
- 一个小而准确的细节，比一整套装饰更有记忆点。
- 技术感可以藏在结构、符号和排版细节里，不必外露成赛博风格。

### 3.2 字体组合

研究文本提到，Anthropic 的品牌字体组合来自 Commercial Type 的 Styrene 与 Klim 的 Tiempos。这个组合兼具技术精致与编辑趣味。

对个人网站的启发：

- 可以使用“清晰 sans-serif + 编辑感 serif + 少量 mono”的组合。
- Sans 负责现代、清楚、界面化。
- Serif 负责人文、杂志感、思考感。
- Mono 负责状态、日期、标签和轻微的工具感。

### 3.3 暖色调色彩系统

Geist 为 Anthropic 开发了一套既适合营销沟通，也能延伸到产品界面的暖色调系统。Abduzeedo 将其概括为“干净的排版、柔和的色调和模块化布局”。

对个人网站的启发：

- 颜色系统要能同时服务首页、作品、文章和小型产品界面。
- 米色、纸色、琥珀色、暖陶土色可以成为基础。
- 模块化布局要清晰，但不要变成冰冷的网格系统。

### 3.4 三个关键词

Geist / 相关评论中提到 Anthropic 品牌围绕三个关键词展开：

- 信任
- 协作
- 清晰度

在个人首页中应转译为：

- 信任：不夸大自己，不堆砌虚假成就数字，不用夸张视觉效果掩盖内容空洞。
- 协作：页面像是在邀请别人进入你的思考，而不是把别人推进营销漏斗。
- 清晰度：每个区块只承担一个任务，让访客快速知道你是谁、你在想什么、你做过什么、如何联系你。

## 4. Claude 界面的视觉语言

Claude 的界面不是单纯聊天窗口，而是把“编辑间”“笔记本”“创意工具”和“artifact 工作区”的体验融合进了对话界面。

它给人的感觉不是“一个很酷的 AI 产品”，而是“一个可以安静工作的思考空间”。

### 4.1 温暖、自然的色调

Claude 设计团队希望界面摆脱冰冷的数码感，因此大量使用柔和米色、琥珀色、暖陶土色。研究文本还提到，Claude 早期界面使用柔和米色调、手绘插图和衬线字体，目的是让软件“感觉像手工制作的”。

对个人网站的启发：

- 背景应该像温暖纸张，而不是冷白屏幕。
- 强调色要像陶土或琥珀，不要像荧光按钮。
- 可以加入非常轻微的手作痕迹，比如不完全机械的线条、纸张颗粒、柔和插图语言。

### 4.2 杂志式排版与大面积留白

Claude 和 Anthropic 的设计都重视分层与排版，界面更像一个可以长时间阅读的杂志版面。大面积留白和缓慢节奏，为长篇阅读创造更好的体验。

对个人网站的启发：

- 首页不是越满越好。
- 留白是设计材料，不是浪费空间。
- 项目、文章、当前思考都应该有清晰标题、小结、列表和元信息。
- 让访客可以慢慢读，而不是被信息轰炸。

### 4.3 颜色与装饰的克制

Claude 的颜色使用非常节制。橙色或暖棕色只出现在少数按钮、状态指示和重点位置，其余主要依靠文字、留白和浅色分隔。

对个人网站的启发：

- 橙色越少越有力量。
- 不要给每个模块都加高亮色。
- 装饰应该帮助理解，而不是证明页面“有设计”。
- 大部分时间，排版、间距、对齐和内容才是主角。

### 4.4 微妙动画与交互

研究文本提到 Claude 使用手绘星号 `spark` 作为状态标识：聆听时静止，回答时微微颤动。这个细节让系统显得在倾听，也让界面更有生命感。

对个人网站的启发：

- 可以设计一个很小的“alive”状态，比如当前在线、正在写作、最近更新、当前关注。
- 动效要像呼吸，不要像表演。
- 小而准确的动画比全页面炫技更有亲和力。
- 动效可以表达“我在这里、我在思考、这个空间是活的”。

### 4.5 功能优先于装饰

Claude 的导航栏和侧边栏保持极简，主要空间让位于对话和 Artifact 区域。提示信息、风险警告、引用链接都用微妙样式呈现，而不是巨大横幅。

对个人网站的启发：

- 导航要克制，内容要成为主角。
- 重要提示可以用标签、细边框、浅底色处理。
- 不要用巨大 banner 展示小信息。
- 项目和文章可以像 artifact 一样呈现：标题、摘要、状态、上下文、下一步。

## 5. 为什么这样设计

### 5.1 传达信任与安全

Anthropic 的使命是构建安全可靠的 AI。视觉上，温暖色调、柔软插图、手作痕迹和充足留白，会营造安全、宁静的氛围。

对个人网站来说，这意味着：

- 访客进入页面时，应该感觉“这里可以慢慢读”。
- 设计不需要急着推销自己。
- 手工感和纸质感能把一点信任还给用户。

### 5.2 表达人本与协作

Anthropic 重视协作和高信任环境。Claude 的界面也鼓励用户把 AI 视为伙伴，而不是支配者。

对个人网站来说，这意味着：

- 页面应该像一个邀请，而不是一个宣传单。
- 你展示的不只是成果，也包括问题、过程和开放线索。
- 访客应该能感觉自己可以与你继续对话。

### 5.3 简洁而可理解

Anthropic 推崇简单有效的方法。设计上应该优先用层级、间距、对齐和排版组织信息，而不是依赖花哨装饰。

对个人网站来说，这意味着：

- 每个区块都要有明确任务。
- 复杂经历要被整理成易读结构。
- 好看不是目标，可理解才是目标；好看应该来自清晰。

### 5.4 差异化与品牌定位

很多 AI 产品视觉缺少人味，容易滑向高科技模板。Anthropic / Claude 选择纸质杂志感、温暖色彩和手工触感，与竞争对手形成差异。

对个人网站来说，这意味着：

- 不要做成另一个“未来感个人主页”。
- 用温暖和编辑感创造记忆点。
- 让技术能力通过判断力、写作和作品上下文体现出来。

### 5.5 激发创造与反思

研究文本提到，Claude 设计团队喜欢传统印刷、手工艺术和胶片摄影，希望为软件注入手作温度和触感。设计师 Kim Bost 也强调，信任与安全感会开启创造性的时刻。

对个人网站来说，这意味着：

- 首页应该让人愿意停留、阅读、思考。
- 视觉冲击不应该打断内容。
- 可信、安静、有人味的界面更容易开启真正的交流。

## 6. 首页设计总纲

### 6.1 核心感觉

这个首页应该像：

- 一张有自然光的纸质书桌。
- 一本正在写的研究笔记。
- 一篇杂志人物页。
- 一个安静的 Claude artifact 工作区。
- 一个让技术、写作、设计和个人气质并存的空间。

它不应该像：

- 常见 SaaS 落地页。
- 深色赛博 AI 官网。
- 卡片堆满的作品集模板。
- 过度包装的个人品牌销售页。
- 只有米色但没有结构的“高级感”页面。

### 6.2 设计支柱

- 信任：诚实文字、克制视觉、清楚的工作证据。
- 协作：让访客知道可以如何与你对话、阅读、合作。
- 清晰：每个区块有明确任务，不制造无效复杂度。
- 温暖：米色、纸感、柔和琥珀色、自然材质感。
- 思考感：像一个正在运转的工作台，而不是静态履历。
- 手艺感：排版、间距、动效、边框和细节都要精确。

## 7. 视觉语言

### 7.1 色彩

推荐方向：

- 背景：温暖象牙白、燕麦色、米色、浅纸色。
- 正文：偏棕的近黑色，不用纯黑。
- 次级文字：暖灰、褐灰、陶土灰。
- 强调色：少量陶土橙、琥珀色、burnt orange。
- 辅助色：少量橄榄绿或苔藓绿，用来平衡橙色。

使用规则：

- 橙色只用于主按钮、选中状态、少量标签或细节强调。
- 不要把橙色当成装饰背景到处铺。
- 不使用高饱和霓虹色。
- 优先用明度、字重、间距和布局制造层级，而不是靠颜色堆叠。

### 7.2 字体

推荐组合：

- 大标题：有编辑感的 serif，适合表达个人观点和叙事。
- 正文 / UI：清晰现代的 sans-serif。
- 元信息：少量 mono，用于日期、状态、标签、版本感信息。
- 中文：尽量选择有阅读感的中文 serif 或质量较高的中文排版，而不是完全依赖系统默认。

使用规则：

- 字体重量不宜过多，2 到 3 个字重足够。
- 标题可以大，但不要喊叫。
- 正文行高要宽松，适合阅读。
- 大写字母和字距拉开只用于很小的标签。

### 7.3 材质与纹理

推荐：

- 非常轻的纸张颗粒。
- 柔和的径向暖光。
- 低透明度发丝边框。
- 像纸张浮起一样的轻阴影。
- 若使用网格，只能非常淡，作为结构暗示。
- 少量手作痕迹，例如轻微不规则线条或插图细节。

避免：

- 玻璃拟态过重。
- 大量发光。
- AI 粒子、抽象光球、随机 3D 装饰。
- 很明显的 mesh gradient。
- 过多图标装饰。

## 8. 布局原则

### 8.1 杂志式留白

页面应该敢于留白。

为什么：

- 留白会让内容显得经过选择，而不是全部倒出来。
- 留白能让访客放慢速度，进入阅读状态。
- Anthropic 的品牌感很大程度来自这种从容。

规则：

- 一个视口尽量只表达一个核心想法。
- 区块数量少一点，但每个区块更有分量。
- 不要因为“空”就补卡片、图标或装饰。
- 项目展示不必全部等权，应该有主次。

### 8.2 清晰层级

每个区块都应该回答一个问题：

- 你是谁？
- 你现在在关注什么？
- 你做过什么？
- 你写了什么？
- 别人可以如何与你产生连接？

原则：

- 用标题、小结、列表、引用、代码块和状态标签组织内容。
- 用间距和对齐建立秩序。
- 让复杂信息可浏览、可理解、可继续深入。

### 8.3 推荐首页结构

建议首页节奏：

- Hero：一句个人 thesis，加一个安静的主要行动。
- Now：当前关注的问题、正在做的项目、近期思考。
- Selected Work：2 到 3 个精选项目，不要做成普通项目网格。
- Writing：文章、笔记、观察或 fragment。
- Collaborate：你欢迎什么样的合作、对话或联系。
- Footer：克制的链接、小型索引、更新时间。

为什么：

- 这个结构比“自我介绍 / 技能 / 项目 / 联系”更有个人气质。
- 它让页面像一个活的思考空间，而不是静态简历。
- 它同时服务读者、潜在合作者和第一次认识你的人。

## 9. 组件语言

### 9.1 导航

建议：

- 顶部导航 3 到 5 个链接。
- 一个主要联系入口即可。
- 导航可以 sticky，但视觉存在感要低。
- logo / 名字不要做复杂动效。

为什么：

- Claude 把空间留给对话和 artifact。
- Anthropic 的导航负责定向，而不是抢注意力。
- 个人网站里，导航永远不应该比内容更重要。

### 9.2 Hero

Hero 不要写成销售页口号，而要像编辑导语。

建议结构：

- 小号 mono eyebrow：身份、状态、地点或当前关注。
- 大标题：一句有人味、具体、有判断的个人表达。
- 简短说明：2 到 3 行，说明你做什么、关心什么、如何工作。
- 一个主按钮：看作品 / 读笔记 / 联系。
- 一个次级链接：开始对话 / 查看 writing。
- 右侧可以放一个 artifact 面板：当前思考、项目摘录、工作日志。

### 9.3 卡片和面板

卡片应该像纸质笔记，而不是 SaaS 功能卡。

规则：

- 边框轻，背景柔，阴影非常克制。
- 卡片数量少，留出主次。
- 标题要像编辑标题，而不是功能名。
- 不要给每张卡都配图标。
- 用状态、日期、类型标签辅助理解即可。

### 9.4 Claude 式 artifact 模块

可以设计一个类似 artifact 的个人模块，但不要做成聊天复制品。

可用形式：

- `Current focus`：当前关注的问题。
- `Working note`：一段正在形成的想法。
- `Prototype log`：近期实验。
- `Reading trace`：读书 / 文章摘录。
- `Open thread`：欢迎别人继续聊的话题。
- `Next question`：当前项目还在追问什么。

为什么：

- 这会让首页有“正在发生”的感觉。
- 它比静态作品集更能体现思考方式。
- 它自然连接 Claude 的协作感。

## 10. 动效与交互

动效应该像呼吸，不像表演。

建议：

- 页面加载时轻微 fade + rise。
- 区块进入视口时轻微 stagger。
- 卡片 hover 时只有很小的浮起或边框变化。
- 文字链接用下划线、颜色或背景轻微变化。
- sticky 导航滚动后略微提高不透明度。
- 可加入非常小的 spark-like 状态点，表示“当前空间是活的”。

规则：

- 动效只服务层级、状态和节奏。
- 不要每个元素都动。
- 必须尊重 `prefers-reduced-motion`。
- 不要让动效打断阅读。

## 11. 文案语气

语气应该清晰、温暖、具体、有判断。

应该做到：

- 说清楚你真正关心什么。
- 用具体项目和问题代替抽象形容词。
- 保持自信，但不夸张。
- 有邀请感，但不销售化。
- 让技术读者和非技术读者都能理解。

避免：

- “Passionate about building innovative solutions.”
- “探索科技与创意的交叉点”但没有具体内容支撑。
- “改变世界”的空泛表达。
- 大量 buzzword。
- 过度包装个人形象。

## 12. 可执行首页方案

### 12.1 首屏建议

内容结构：

- Eyebrow：`Personal site / writing / interfaces / experiments`
- Headline：一句关于你如何思考、设计、构建的强表达。
- Body：2 到 3 行，具体说明你的兴趣和工作方式。
- Primary CTA：`Read the notes` 或 `View selected work`
- Secondary CTA：`Start a conversation`
- Side artifact：当前关注、最近笔记、项目状态或小型工作日志。

设计原因：

- 先建立人物气质和可信度，再引导点击。
- 把个人网站从“作品陈列”变成“思考入口”。
- 用 artifact 模块引入 Claude 式协作空间感。

### 12.2 内容区建议

建议顺序：

- `Now`：当前问题和近期关注。
- `Selected Work`：少量精选项目，每个项目说明背景、判断和结果。
- `Writing`：文章、笔记、观察、读书摘录。
- `Collaborate`：你欢迎的合作方式或交流主题。
- `Footer`：链接、更新时间、简短签名。

设计原因：

- 先展示当下的思考，再展示过去的作品。
- 作品不只是结果，而是体现判断力。
- 写作区增加可信度和持续性。

## 13. 当前项目可沿用的设计 token

项目里已有的 `tokens.css` 方向非常接近这个设计语言，可以继续沿用：

```css
:root {
  --paper: #f8f3ea;
  --paper-soft: #fbf8f2;
  --paper-strong: #efe4d4;
  --ink: #181511;

```

为什么：

- 这组颜色已经具备 Anthropic 式的纸色、墨色、陶土橙和自然绿色。
- serif / sans / mono 的组合也适合杂志感 + 工具感 + 思考感。
- 后续首页实现不需要推翻现有视觉系统，只需要更严格地控制层级、留白和组件密度。

## 14. Do / Don't

Do:

- 用留白建立信任。
- 用排版塑造个性。
- 让橙色保持稀有。
- 展示当前思考和过程。
- 让每个区块都有明确任务。
- 让页面安静到适合阅读。
- 用微小手作细节增加人味。

Don't:

- 不要过度使用橙色。
- 不要加入 AI 粒子、霓虹、赛博装饰。
- 不要把所有项目做成等权卡片。
- 不要用巨大 banner 呈现小提示。
- 不要让每个元素都动起来。
- 不要直接复制 Claude 聊天气泡。
- 不要把 Anthropic 风格误解成“米色 + 极简”而已。
